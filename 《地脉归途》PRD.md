## 1. 产品概述

### 1.1 产品定位
地脉归途是一款结合LBS行走记录、情绪日记与艺术创作的个人成长与疗愈App。它将用户的真实脚步转化为一张独一无二的、可视化的心灵地图，帮助用户在行走中找回真正的自己。

### 1.2 核心价值主张
**"所有的出发，都是为了回家"**

- **行走记录**: 将所有出行方式（步行/骑行/驾车/飞行）转化为艺术化的"能量线"
- **情绪日记**: 在轨迹上添加"心绪节点"，记录思绪与感受
- **世界探索**: 发现全球的"圣迹"，连接内在世界与地球能量网络
- **内在成长**: 通过"先行者"的引导，在对话中找到真正的自己

### 1.3 产品愿景
成为用户首选的、通过行走找回自我的数字伙伴。每一次出发都是为了更好地回归，每一段旅程都是通往内心的归途。

## 2. 目标用户

### 2.1 主要用户画像

#### 画像1: 都市减压者
- **年龄**: 25-40岁
- **特征**: 城市白领，工作压力大，希望通过行走整理思绪
- **痛点**: 感到职业倦怠，缺乏内心平静
- **诉求**: 需要一个工具帮助记录和回顾自己的心路历程

#### 画像2: 创意记录者
- **年龄**: 20-35岁
- **特征**: 喜欢记录生活，但觉得传统日记形式单调
- **痛点**: 灵感转瞬即逝，难以有效捕捉
- **诉求**: 能将行走与记录结合的艺术化工具

#### 画像3: 精神探索者
- **年龄**: 28-45岁
- **特征**: 对正念、冥想和自我探索感兴趣
- **痛点**: 理论多于实践，难以付诸行动
- **诉求**: 有引导的、实践性的成长体验

### 2.2 用户规模预估
- **全球正念/冥想App市场**: 超过3亿活跃用户
- **参考产品**: Headspace/Calm等头部正念App
- **目标**: 第一年获取50万注册用户，5万DAU

## 5. 功能详细需求

### 5.1 心灵地图系统

#### 5.1.1 GPS轨迹记录
**需求描述**: 记录用户所有类型的移动轨迹

**支持的出行方式**:
| 方式 | 速度区间 | 自动识别 |
|------|----------|----------|
| 步行/慢跑 | <10 km/h | 是 |
| 骑行/电动车 | 10-30 km/h | 是 |
| 驾车/火车 | 30-120 km/h | 是 |
| 高铁/飞机 | >120 km/h | 是 |

**功能要求**:
- 后台稳定记录GPS轨迹
- 根据速度自动判断出行方式
- 结束后允许用户手动校准
- 支持随时开始/结束（无需闭合）
- 低功耗设计

**技术方案**:
- CoreLocation后台定位
- 智能采样算法（根据速度调整采样频率）

#### 5.1.2 能量线渲染
**需求描述**: 将GPS轨迹美化为艺术化的"能量线"

**不同出行方式的视觉特征**:
| 方式 | 线条特征 | 颜色 |
|------|----------|------|
| 步行 | 坚实、有韵律的脉冲光线 | 琥珀色/暗金色 |
| 骑行 | 流畅、连续的流光 | 青色/浅绿色 |
| 驾车 | 疾速、半透明的光带 | 银白色/淡蓝色 |
| 飞行 | 飘渺、高悬的以太弧光 | 紫色/水晶色 |

**技术方案**:
- Metal/Core Animation渲染
- 自定义着色器实现光效

#### 5.1.3 心绪节点
**需求描述**: 在轨迹上添加情绪和思绪记录

**节点内容类型**:
- **情绪标签**: 从预设的情绪图标/色彩中选择
- **文字记录**: 最多500字
- **语音胶囊**: 最多3段，每段2分钟内
- **图片**: 最多9张
- **视频**: 1段，1分钟内

**功能要求**:
- 点击轨迹任意位置添加节点
- 节点与轨迹位置精确绑定
- 支持增、删、改、查

#### 5.1.4 心灵地图可视化
**需求描述**: 在主界面以可交互地图展示所有记录

**功能要求**:
- 支持平移、缩放
- 显示所有能量线和心绪节点
- 按时间筛选显示
- 地图加载时间<3秒
- 100条轨迹数据时平滑流畅

**地图主题**:
- 默认主题（免费）
- 高级主题（订阅解锁）：水彩风、星空风、赛博朋克风等

### 5.2 圣迹系统

#### 5.2.1 三层圣迹体系
**需求描述**: 建立分层的地脉节点体系

| 层级 | 名称 | 定义 | 图标风格 |
|------|------|------|----------|
| Tier 1 | 源点圣迹 | 全球性传说级地标（金字塔、巨石阵等） | 动态旋转的光构几何体 |
| Tier 2 | 地脉节点 | 区域性重要景观（城市主山、大型公园等） | 精致的静态图标（山峰/河流/古建筑） |
| Tier 3 | 心绪锚点 | 社区级宁静地点（小公园、林荫道等） | 柔和的光点/涟漪 |

**数据来源**:
- Tier 1: 官方预设，极其稀有
- Tier 2: 官方预设 + 用户提名审核
- Tier 3: 官方批量预设 + 用户自定义

#### 5.2.2 星脉图视图
**需求描述**: 艺术化的世界地图展示

**视觉设计**:
- 类似古老羊皮卷/星象图的风格
- 地球主脉以微光线条绘制
- 圣迹标记为闪亮的"星辰"

**功能要求**:
- 点击星辰弹出圣迹卡片
- 支持缩放和拖动
- 显示圣迹名称和缩略图

#### 5.2.3 圣迹录列表
**需求描述**: 列表形式展示所有圣迹

**功能要求**:
- 按大洲分类
- 每项包含：缩略图、名称、地区、一句话描述
- 支持搜索和排序

#### 5.2.4 圣迹详情页
**需求描述**: 展示圣迹的完整信息

**内容结构**:
- 沉浸式头图/视频
- 名称与地点
- 地脉解读（官方撰写的灵性文字）
- 历史与传说
- 「回响」区域（UGC内容）
- 「规划朝圣之旅」按钮

#### 5.2.5 旅程规划器
**需求描述**: 提供多模式出行方案的"意义解读"

**出行方式对比**:
| 方式 | 旅程解读（优点） | 行前思考（缺点） |
|------|------------------|------------------|
| 飞机 | 超越凡尘，俯瞰大地 | 与地脉短暂失联 |
| 高铁 | 贴地飞行，流动冥想 | 掠过细节，不易停留 |
| 火车 | 深入肌理，感受脉搏 | 时间的臣服 |
| 自驾 | 自由意志，探索未知 | 心力的投入 |
| 步行 | 一步一印，身心合一 | 肉身的考验 |

**功能要求**:
- 调用第三方地图API估算距离/时间
- 展示预设的"意义解读"文本
- 提供跳转按钮调用外部导航App

#### 5.2.6 寻迹申请
**需求描述**: 用户提名新的圣迹

**申请流程**:
1. **精准定位**: 在地图上放置探针获取经纬度
2. **命名定义**: 输入名称和一句话描述
3. **阐述理由**: 历史传说、个人体验、地貌特征
4. **提供佐证**:
   - 实地照片/视频（必需，至少3张）
   - 文献资料截图（强烈建议）
5. **提交承诺**: 勾选真诚分享的承诺

**审核标准**:
- 可验证性：能找到文献支撑
- 独特性：与已有圣迹有明显区别
- 社群共鸣：多用户提名相同地点

**审核结果**:
- 通过：进入发布流程
- 驳回：给出清晰、敬意的理由

**发布流程**:
- 星脉图上出现"脉动星云"24小时
- 全局推送通知
- 提名者获得"最初的寻迹者"称号

### 5.3 回响系统

#### 5.3.1 UGC内容发布
**需求描述**: 用户在圣迹页留下感受

**内容形式**:
- 文字：最多500字
- 媒体：1个视频(1分钟) / 3段音频(2分钟) / 9张图片

**发布者显示**:
- 显示昵称和头像
- 或选择"匿名行者"

**隐私设置**:
- 默认"仅自己可见"
- 可切换为"众生可见"

**"零回响"状态**:
- 显示引导文字："万籁俱寂，此地的第一个回响，正等待你的声音……"

#### 5.3.2 内容审核
**需求描述**: 防止不当内容

**审核机制**:
- 公开内容需进入后台审核队列
- 敏感词过滤
- 人工审核

### 5.4 意向系统

#### 5.4.1 意向标记
**需求描述**: 匿名、无交互的共鸣系统

**功能流程**:
1. 用户点击"我亦向往"按钮
2. 选择计划前往的月份
3. 确认后完成意向标记

**数据处理**:
- 后台记录 (site_id, year, month, user_count)
- 用户点击后 user_count +1
- 仅显示聚合数字，无任何用户信息交换

**展示方式**:
- "包括你在内，已有 X 位行者，向往于Y月抵达此地。"

### 5.5 引路系统

#### 5.5.1 先行者管理
**需求描述**: 官方认证的引导者

**先行者资格**:
- 官方邀请认证
- 现实世界的专家（正念导师、徒步者、诗人等）
- 能深刻表达行走感悟

**展示方式**:
- 「引路」标签页
- 展示先行者卡片（肖像、名字、简介）

#### 5.5.2 星图合集
**需求描述**: 先行者的行走系列

**星图结构**:
- 有主题的多次行走系列
- 例如："重走长安古道"、"城市静谧角落"
- 视觉上像星座，行走是连线，节点是星辰

#### 5.5.3 共鸣行走
**需求描述**: 体验先行者的路线

**功能流程**:
1. **预览**: 在地图上显示"光之路"（半透明发光轨迹）
2. **聆听**: 点击节点预览先行者的心绪记录
3. **开始共鸣**: 进入特殊行走模式
4. **情景音频触发**: 接近先行者节点时自动播放音频
5. **创造自己的回响**: 同时生成自己的能量线和节点

#### 5.5.4 强制反思
**需求描述**: 引导用户从模仿走向对话

**机制设计**:
- 共鸣行走结束后弹出反思界面
- 左侧显示先行者的心绪记录
- 右侧显示："先行者在此感受到了'XX'，而你呢？"
- 用户必须留下自己的心绪节点才能保存

## 8. 技术架构

### 8.1 技术栈

**前端**:
- 语言: Swift 5.9
- UI框架: SwiftUI
- 地图: MapKit
- 定位: CoreLocation
- 音频: AVFoundation
- 图形: Metal/Core Animation
- 支付: StoreKit 2
- 本地存储: SwiftData

**后端**:
- BaaS: Supabase
  - 数据库: PostgreSQL + PostGIS
  - 认证: Supabase Auth
  - 实时推送: Supabase Realtime
  - 存储: Supabase Storage

**第三方服务**:
- 地图API: Google Maps / 高德地图

### 8.2 数据库设计

**核心表结构**:
- `users`: 用户信息
- `tracks`: 轨迹数据（PostGIS geometry）
- `nodes`: 心绪节点
- `sacred_sites`: 圣迹数据
- `echoes`: 回响内容
- `intentions`: 意向记录
- `guides`: 先行者信息
- `constellations`: 星图合集
- `nominations`: 寻迹申请
